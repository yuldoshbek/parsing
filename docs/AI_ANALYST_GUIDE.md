## 📦 ЧТО ТЫ ПОЛУЧАЕШЬ НА ВХОД

Тебе дают CSV или JSON-файл с отзывами. Каждая строка — один отзыв.

**Поля в данных:**
| Поле | Пример | Что это |
|---|---|---|
| `product_id` | 246290927 | Артикул WB |
| `brand` | Give Creative | Бренд |
| `name` | 3d ручка для девочек | Название товара |
| `sku_variant` | Розовый / PCL | Конкретный вариант (цвет/тип пластика) |
| `rating` | 1-5 | Оценка покупателя |
| `rating_group` | позитив / нейтрал / негатив | Уже посчитано парсером |
| `pros` | Лёгкая, не греется | Достоинства (из отзыва) |
| `cons` | Засоряется | Недостатки |
| `text` | Ребёнок доволен, но… | Свободный комментарий |
| `working_base` | 64 | Рабочая база для % (знаменатель) |
| `positive_pct` | 67.2 | % позитивных от рабочей базы |
| `neutral_pct` | 9.4 | % нейтральных (3★) от рабочей базы |
| `negative_pct` | 23.4 | % негативных от рабочей базы |

## 🎯 ТВОЯ ЗАДАЧА: ЧТО НУЖНО ПОСТРОИТЬ

### БЛОК 1. Воронка фильтрации (уже в данных, проверь)
Убедись что для каждого артикула есть:
- `total_parsed` — сколько спарсили
- `no_text` — без текста (только звёзды, мусор)
- `working_base` = `total_parsed` - `no_text` — рабочая база

**Это знаменатель для всех процентов. Никогда не считай % от `total_parsed` или от числа всех отзывов на ВБ.**

---

### БЛОК 2A. «КТО» — кто покупает / кому покупают

> ⚠️ Это отдельный блок. Не смешивай с блоком «КОГДА».

Пройди по текстам отзывов и ответь: **кто является получателем товара?**

**Категории** (ищи упоминания в тексте, pros, cons):

| Категория | Маркеры в тексте |
|---|---|
| Родители → ребёнку | «ребёнку», «сыну», «дочери», «купила ребёнку», «дочке» |
| Бабушки/дедушки → внукам | «внуку», «внучке», «внучке подарили» |
| Ребёнок сам себе | «купил сам», «себе», «сам пользуюсь» |
| Взрослый → ребёнку (общее) | «подарили ребёнку», «детям», «детский» |
| Взрослый для себя | «сам рисую», «сама занимаюсь», «для творчества себе» |

**Что посчитать:**
- Сколько отзывов относится к каждой категории (абс. число + % от working_base)
- Если категория не определяется — записывай в «Прочее»
- Сумма всех % **не должна** быть 100% — один отзыв может попасть в несколько категорий

**Пример вывода:**
```
КТО ПОКУПАЕТ / КОМУ:
  Родители → ребёнку:      47 отз.  = 73.4%
  Бабушки/дедушки → внукам: 8 отз.  = 12.5%
  Взрослый для себя:         5 отз.  =  7.8%
  Прочее/не определено:      4 отз.  =  6.3%
```

---

### БЛОК 2B. «КОГДА» — повод / событие покупки

> ⚠️ Это отдельный блок. Не смешивай с блоком «КТО».

Пройди по текстам и найди: **по какому поводу куплен товар?**

**Категории:**

| Категория | Маркеры в тексте |
|---|---|
| Новый год | «новый год», «НГ», «под ёлку», «дед мороз», «рождество» |
| День рождения | «день рождения», «ДР», «на ДР», «5 лет», «10 лет» |
| 8 марта | «8 марта», «на 8 марта» |
| 23 февраля | «23 февраля», «на 23» |
| 1 сентября | «1 сентября», «к школе» |
| Просто в подарок (повод не указан) | «в подарок», «подарок» — без конкретной даты |
| Покупка для себя (без повода) | «сам купил», «себе», нет упоминания подарка |
| Прочее | всё остальное |

**Что посчитать:**
- Сколько отзывов по каждому поводу (абс. число + % от working_base)
- Один отзыв = один основной повод (если упомянут Новый год — пиши «Новый год», даже если также написано «в подарок»)

**Пример вывода:**
```
КОГДА / ПОВОД:
  Новый год:          24 отз.  = 37.5%
  День рождения:      18 отз.  = 28.1%
  Просто в подарок:    9 отз.  = 14.1%
  8 марта:             4 отз.  =  6.3%
  Для себя:            5 отз.  =  7.8%
  Прочее:              4 отз.  =  6.3%
```

### БЛОК 3. Боли рынка (из негатива 1-3★)

Проанализируй отзывы с `rating_group = "негатив"` (это 1★, 2★ и 3★).

**Найди повторяющиеся боли.** Для каждой боли:
- Название проблемы (коротко)
- Сколько отзывов упоминают (абс. + % от `working_base`)
- Краткое описание сути проблемы
- 1-2 цитаты из реальных отзывов

**Частые боли (проверь наличие):**
- Засорение сопла / пластик не выходит
- Несовместимость пластика (только PCL, нельзя другой)
- Быстрая поломка (перестала работать за 1-2 дня)
- Поломка при первом использовании
- Непонятная инструкция
- Нагрев / ожоги
- Проблемы с зарядкой / кабелем
- Помятая упаковка

---

### БЛОК 4. Конструктивный негатив из позитивных отзывов (4-5★)

> 💡 Это важный и отдельный блок. Его часто пропускают — не пропускай.

**Суть:** Люди ставят 4-5★, но в тексте упоминают что-то, что им не понравилось. Такой фидбек — самый ценный, потому что человек настроен конструктивно.

**Что искать:** В отзывах с `rating = 4` или `rating = 5` ищи семантику недовольства:
- слова «но», «однако», «хотя», «минус», «жаль», «мог бы», «хотелось бы»
- конкретные жалобы в поле `cons` при `rating >= 4`
- замечания типа «всё хорошо, но коробка помятая»

**Что посчитать:**
- Сколько отзывов 4-5★ содержат конструктивный негатив (% от всех позитивных)
- Топ-3 темы, на которые жалуются даже довольные покупатели

**Пример вывода:**
```
КОНСТРУКТИВНЫЙ НЕГАТИВ (из 4-5★ отзывов):
  Отзывов с замечаниями: 12 из 43 позитивных = 27.9%

  Топ-темы замечаний:
  1. Запас пластика мал — быстро заканчивается (7 отз.)
  2. Нет трафаретов нужного типа (4 отз.)
  3. Провод слишком короткий (3 отз.)

  Цитата: «Ручка отличная, не греется, ребёнок доволен.
  Но дали бы больше пластика в комплекте — закончился за неделю.»
```

## ❌ ЧАСТЫЕ ОШИБКИ — НЕ ДЕЛАЙ ТАК

| Ошибка | Правильно |
|---|---|
| Считать % позитив/негатив от `total_parsed` | Считать от `working_base` |
| Суммировать % «КТО» до 100% | Блок «КТО» не суммируется — категории могут пересекаться |
| Складывать «КТО» и «КОГДА» в один блок | Это два разных блока |
| Игнорировать 3★ — «ни позитив, ни негатив» | 3★ = нейтрал, включать обязательно |
| Искать боли только в поле `cons` | Боли бывают в `text`, `pros` и даже в 4-5★ отзывах |
| Анализировать отзывы разных SKU вместе | Если есть поле `sku_variant` — указывай его в анализе |
