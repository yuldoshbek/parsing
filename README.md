# WB Reviews Parser

Парсер отзывов с Wildberries через внутренний API. Без браузера, без антибота.

> **Версия:** v5.0 | **Дата:** апрель 2026 | **Язык:** Python 3.10+

---

## Быстрый старт

```bash
# 1. Установить зависимости
pip install requests

# 2. Добавить артикулы в articles.txt (по одному на строку)
echo "246290927" >> articles.txt

# 3. Запустить
python main.py

# Результаты → папка results/ (создаётся автоматически)
```

**Windows:** двойной клик по `run.bat` — запустит парсер и сохранит лог в `run_err.txt`.

---

## Как это работает

Парсер делает прямые HTTP-запросы к внутреннему WB API — никакого браузера, никакого Playwright.

```
articles.txt
    │
    ▼
card.wb.ru/cards/v4/detail        → brand, name, feedbackCount, root (imtId)
    │
    ▼
feedbacks2.wb.ru/feedbacks/v1/{root}  → все отзывы (до 1000)
    │
    ▼
results/
  ├── data_export.json             ← полный дамп всех артикулов
  ├── data_export.txt              ← читаемый текстовый отчёт
  ├── all_reviews.csv              ← все отзывы, плоская таблица
  └── all_questions.csv            ← вопросы (сейчас пустой — см. ограничения)
```

**Почему не Playwright?**
WB DDoS-Guard блокирует браузеры по поведению и IPv6. Прямые API-запросы не триггерят защиту, работают в 5-10 раз быстрее.

## Конфигурация

### articles.txt

Один артикул WB на строку. Пустые строки и строки с `#` игнорируются.

```
# 3D-ручки Give Creative
246290927
480055076
480055077
```

### Директория вывода

По умолчанию результаты пишутся в `results/` рядом с кодом.  
Чтобы писать в другую папку — задай переменную окружения `WB_RESULTS_DIR`:

```bash
# Linux / Mac
WB_RESULTS_DIR=/data/wb_output python main.py

# Windows CMD
set WB_RESULTS_DIR=C:\Users\User\Desktop\wb_output && python main.py

# Windows PowerShell
$env:WB_RESULTS_DIR="C:\Users\User\Desktop\wb_output"; python main.py
```

Это позволяет держать код и данные в разных папках — рабочий процесс проекта.

## Формат данных

### all_reviews.csv — поля

| Поле | Пример | Описание |
|---|---|---|
| `product_id` | 246290927 | Артикул WB |
| `brand` | Give Creative | Бренд |
| `name` | 3d ручка для девочек | Название товара |
| `sku_variant` | Розовый / PCL | Вариант SKU (из API, поле color) |
| `rating` | 4 | Оценка 1-5 |
| `rating_group` | позитив | позитив / нейтрал / негатив |
| `pros` | Лёгкая, не греется | Достоинства |
| `cons` | Засоряется | Недостатки |
| `text` | Ребёнок доволен, но… | Свободный комментарий |
| `date` | 5 марта, 14:32 | Дата отзыва (МСК) |
| `total_wb` | 156 | Всего отзывов на WB |
| `total_parsed` | 84 | Спарсено с текстом |
| `no_text` | 20 | Без текста (только звёзды) |
| `working_base` | 64 | Рабочая база = знаменатель для % |
| `positive_pct` | 67.2 | % позитивных от working_base |
| `neutral_pct` | 9.4 | % нейтральных (3★) |
| `negative_pct` | 23.4 | % негативных (1-2★) |
| `sum_pct_check` | 100.0 | Самопроверка: должна быть ~100 |

### Воронка данных (funnel stats)

```
total_wb          — все отзывы на WB (из API feedbackCount)
  └─ total_parsed — отзывы с текстом, которые спарсили
       └─ no_text — только звёзды, без текста
       └─ working_base = total_parsed - no_text  ← знаменатель для %
            └─ позитив (4-5★): X%
            └─ нейтрал (3★):   Y%
            └─ негатив (1-2★): Z%
            = 100%
```

**Важно:** все проценты считаются от `working_base`, не от `total_wb` или `total_parsed`.

## Известные ограничения

### 1. Вопросы недоступны

`questions.wb.ru` не резолвится DNS с большинства сетей. `scrape_questions()` возвращает `[]` с WARNING в логе. `all_questions.csv` будет пустым.

**Воркэраунд:** нужен VPN или сервер с другим DNS/IP.

### 2. API-лимит 1000 отзывов

`feedbacks2.wb.ru` возвращает максимум 1000 записей на артикул (самые новые). Для крупных товаров часть отзывов теряется:

| Артикул | Всего на WB | Получено |
|---|---|---|
| 844748724 | 2317 | 1000 |
| 428894982 | 1978 | 1000 |
| 124436959 | 1477 | 1000 |

**Воркэраунд:** разбивка по датам через несколько запросов (пока не реализовано).

### 3. SKU-склейка

WB объединяет несколько вариантов товара (цвет, тип пластика) в одну карточку. Все отзывы склейки доступны через любой артикул из группы. Поле `sku_variant` берётся из поля `color` каждого отзыва — это лучший доступный способ определить вариант SKU.
