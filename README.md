# WB Reviews Parser

Парсер отзывов с Wildberries — два режима работы.

> **Версия:** v5.1 | **Дата:** апрель 2026 | **Язык:** Python 3.10+

---

## Два инструмента

| Файл | Режим | Когда использовать |
|---|---|---|
| `pro_scraper_reviews.py` | Playwright + браузер + PRO-фильтр | Основной парсер — полный пайплайн |
| `main.py` | Прямые HTTP-запросы (без браузера) | Лёгкий режим — быстрый сбор без фильтрации |

---

## Быстрый старт — основной парсер

```bash
# 1. Установить зависимости
pip install playwright playwright-stealth
playwright install chromium

# 2. Добавить артикулы в articles.txt (по одному на строку)
echo "246290927" >> articles.txt

# 3. Запустить
python pro_scraper_reviews.py

# Результаты → папка results_pro_reviews/ (создаётся автоматически)
```

### Быстрый старт — лёгкий режим (без браузера)

```bash
pip install requests
python main.py
# Результаты → папка results/
```

## Как работает основной парсер (pro_scraper_reviews.py)

```
articles.txt
    │
    ▼
[Playwright: открывает wildberries.ru, проходит DDoS-Guard]
    │
    ▼
card.wb.ru/cards/v4/detail?nm={nmId}
    → brand, name, total_wb_nm (оценок у этого nmId), root (imtId), nm_color
    │
    ▼
feedbacks2.wb.ru/feedbacks/v1/{root}   [через page.evaluate — запрос изнутри браузера]
    → все отзывы склейки за 365 дней (skip/take пагинация)
    │
    ▼
[SKU De-gluing: разбивка по nmId → card API → цвет каждого варианта]
    │
    ▼
[PRO-фильтр: 100% негатив + позитив > 17 слов без стоп-фраз]
    │
    ▼
results_pro_reviews/
  ├── {nmId}/                       ← все SKU вместе
  │   ├── PRO_reviews_LLM.txt       ← для нейросети / аналитика
  │   ├── PRO_reviews_LLM.json      ← для IT / автоматизации
  │   └── PRO_reviews_Analytics.csv ← для Excel
  ├── {nmId}_{SKU}/                 ← каждый вариант отдельно
  │   └── ...
  ├── ALL_reviews_Analytics.csv     ← все артикулы в одном файле
  ├── ALL_reviews_LLM.json
  └── ALL_reviews_LLM.txt
```

## SKU De-gluing — расклейка вариантов

WB «склеивает» несколько артикулов (nmId) в одну страницу отзывов.
Парсер разделяет их обратно по полю `nmId` в каждом отзыве (надёжнее `color` — WB часто оставляет `color=''`).

```
root=232715088 → 83 отзыва (все варианты вперемешку)
    ├── nmId=246290927 → "Оранжевый" → 55 отзывов
    └── nmId=371806956 → "FDM"       → 18 отзывов
         + остальные варианты склейки...
```

**Кэш по root:** если два артикула из одной склейки — feedbacks API вызывается один раз,
второй артикул берёт данные из кэша. Нет дублей, меньше нагрузки на WB.

## PRO-фильтр

Оставляет только отзывы с реальной информацией о товаре.

| Правило | Оценка | Логика |
|---|---|---|
| Rule 1 | 1★ 2★ 3★ | Проходит всегда — негатив важен даже в 2 словах |
| Rule 2 | 4★ 5★ | Нужно > 17 слов И отсутствие стоп-фраз |
| Исключение | 4★ 5★ | Слова-брейкеры (`брак`, `сломалась`, `не работает`...) отменяют стоп-фразы |

**Стоп-фразы** — 3 категории:
- «Синдром подарочника»: `ещё не пробовали`, `ждёт дня рождения`, `брали на подарок`...
- «Ода логистике»: `пришло быстро`, `упаковка целая`, `спасибо продавцу`...
- «Белый шум»: `всё супер`, `всё работает`, `ребёнок доволен`, `соответствует описанию`...

Подробное описание с примерами: [`FILTER_PIPELINE_EXPLAINED.txt`](FILTER_PIPELINE_EXPLAINED.txt)

## Воронка данных

```
На карточке WB:   110   (total_wb_nm — оценок у этого nmId)
                   51   «немые» ★ — API не возвращает, нет способа достать
  ─────────────────
В API (все форматы): 55  (feedbacks с текстом + без текста)
                  - 28   старше 365 дней
  ─────────────────
Спарсено (год):   27   (total_parsed)
                  -  3   без текста (только ★)
  ─────────────────
Рабочая база:     24   (working_base — знаменатель для %)
                  - 16   отсеяно PRO-фильтром
  ─────────────────
После фильтра:     8   ← финальный набор для анализа (after_filter)
```

**Все % считаются от `after_filter`** (text_base = after_filter с текстом), сумма = 100%.
`sum_pct_check` в каждом файле — самопроверка округления.

## Поля CSV

| Поле | Описание |
|---|---|
| `product_id` | Артикул WB (nmId) |
| `brand` | Бренд |
| `name` | Название товара |
| `sku_variant` | Вариант SKU (цвет, комплектация) |
| `total_wb_nm` | Оценок у этого nmId на WB (per-SKU, точный) |
| `total_wb_root` | Оценок всей склейки (feedbackCount root, справочно) |
| `total_parsed` | Спарсено за период |
| `no_text` | Без текста (только ★) |
| `working_base` | С текстом до PRO-фильтра |
| `after_filter` | Прошли PRO-фильтр |
| `text_base` | С текстом после фильтра (знаменатель для %) |
| `positive_pct` | % позитив (4-5★) от text_base |
| `neutral_pct` | % нейтрал (3★) от text_base |
| `negative_pct` | % негатив (1-2★) от text_base |
| `sum_pct_check` | Самопроверка: должна быть 100.0 ± 0.2 |
| `date` | Дата отзыва (МСК, UTC+3) |
| `rating` | Оценка 1–5 |
| `rating_group` | позитив / нейтрал / негатив |
| `word_count` | Кол-во слов в отзыве |
| `pros` / `cons` / `text` | Текст отзыва |
| `seller_answer` | Ответ продавца |

## Конфигурация (pro_scraper_reviews.py)

```python
DAYS_BACK           = 365      # глубина сбора в днях
MIN_WORDS_POSITIVE  = 17       # минимум слов для 4-5★
HEADLESS            = False    # False = видимый браузер (лучше для DDoS-Guard)
RESULTS_DIR         = "results_pro_reviews"
```

Также через переменную окружения:
```bash
WB_RESULTS_DIR=C:\my_output python pro_scraper_reviews.py
```

## Известные ограничения

### 1. Немые оценки недоступны
WB позволяет ставить ★ без текста. Такие оценки feedbacks API не возвращает.
Для артикула 246290927: 110 оценок на карточке, 55 доступны через API.

### 2. API-лимит 1000 отзывов
`feedbacks2.wb.ru` возвращает максимум 1000 записей. Для крупных товаров берём только первые 1000 (самые новые):

| Артикул | На WB | Доступно |
|---|---|---|
| 844748724 | 2317 | 1000 |
| 428894982 | 1978 | 1000 |

### 3. Вопросы недоступны
`questions.wb.ru` не резолвится DNS с большинства сетей. `scrape_questions()` возвращает `[]`.
